---

Evaluate the user's game for polish and aesthetic coherence. Read the framework in `references/framework.md` for the full evaluation model, and check `references/gotchas.md` for common analysis mistakes to avoid.

The core principle: consistency beats quality. Identity comes from rules applied uniformly, not from production value. A game with deliberately limited visuals applied consistently reads as higher quality than a game with mixed fidelity.

Audio and visual direction are the same job with different tools. Evaluate them as a partnership -- both should derive from the same theme and constraints. Solving them separately produces elements that are individually good but don't compound.

For each relevant dimension (visual identity, audio design, typography, game feel), assess what is working and what is not. Be direct and specific. End with the 1-2 highest-leverage changes.

Judge the design on its own terms. Not every game needs the same level of visual fidelity or audio complexity.
