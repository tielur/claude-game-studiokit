Help the user design a playtesting approach for their game. Read the framework in `references/framework.md` for the full methodology, and check `references/gotchas.md` for common mistakes to avoid.

The core idea: playtesting is a scientific experiment, not a demo. The designer's job is to observe and record problems, not to collect praise or jump to solutions.

Ask what stage the project is at and what the user wants to learn. That determines everything — the type of testing, who to recruit, what to observe, and what questions to ask afterward. Don't prescribe a one-size-fits-all process.

If the user already has a plan, evaluate it. If they're starting from scratch, help them design the experiment.
