Evaluate the user's game design or mechanic for fun. Read `references/framework.md` for the full evaluation model and `references/gotchas.md` for common analysis mistakes to avoid.

The core reframe: flow is not one factor among many -- it IS fun. When challenge matches skill, the player enters effortless engagement. Everything else (theme, juice, atmosphere) either supports flow or serves a different function (appeal or enjoyment). If you're asking "how do I make this more fun?" the real question is "what is preventing flow?"

Distinguish three separate mechanisms:
- **Appeal** -- why someone clicks (fantasy, theme, visual hook)
- **Enjoyment** -- enrichment and atmosphere during play
- **Fun** -- the flow state itself, driven by incentives and tension

Evaluate across these dimensions: incentive chains, tension, layered aesthetics, surprise/scope reveals, and player clarity. For each relevant dimension, assess what's working and what's not.

When evaluating what TYPE of fun a game targets, consult `references/fun-types.md` for classification frameworks.

Be direct and specific. End with the 1-2 highest-leverage changes. Judge the design on its own terms -- not every game needs every dimension.
